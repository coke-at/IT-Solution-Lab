# IT Solution Lab

IT Solution Lab 是一个面向企业 IT、安全运维和解决方案售前成长的个人学习实验室。项目将产品知识、交付经验、故障排查、厂商研究和方案设计整理为可查询、可比较、可持续维护的 Web 工作台。

## 项目目标

- 建立网络、安全、计算、存储、云与基础软件的结构化知识体系。
- 将设备知识从简单参数罗列转化为场景、部署、验收和排障方法。
- 记录从企业安全运维到解决方案工程师的学习路线与作品证据。
- 为产品研究、厂商对比、实验复盘和解决方案输出提供统一模板。

## 技术栈

- HTML5、CSS3、原生 JavaScript（ES Modules）
- Browser Local Storage，用于保存工单、询价和手册版本等本地数据
- Node.js 构建脚本，用于生成 `dist/client`、`dist/server` 和托管配置
- Fetch Worker 形式的静态资源服务入口
- Git / GitHub 版本管理

项目没有引入前端框架或第三方运行时依赖。

## 当前完成内容

- IT Solution Lab 首页、侧边导航和响应式页面结构
- 技术能力地图与个人成长路线
- AF、aTrust、EDR、SIP、HCI、AC 六个统一产品研究模板
- 企业出口安全场景的厂商路线对比示例
- 解决方案中心和成长型企业安全与云基础设施升级案例
- 动手实验室、实验步骤和复盘框架
- 个人成长 Dashboard 与阶段性自评展示
- 设备知识库，支持关键词搜索、厂商/类别筛选、排序和加载更多
- 最多 4 个产品的横向比较
- 产品详情中的能力、部署交付、验收和故障排查内容
- 厂商能力图谱、交付标准和排障手册查询
- 用户手册与版本登记中心，支持本地保存核验记录
- 本地工单、报价、询价和招投标文档工具
- 明暗主题、本地状态保存和移动端导航适配
- Open Graph 图片及静态站点构建、托管配置

## 未完成内容

- 六个产品已建立基础模板，但具体型号、版本、授权、配置命令和真实项目证据仍需持续深化。
- 厂商路线对比仍需扩展奇安信、绿盟、天融信等厂商和更多客户场景。
- 实验室目前以流程框架和示例为主，真实实验记录、截图、配置和验证结果待沉淀。
- 成长 Dashboard 使用静态自评基线，尚未与实际作品和学习记录自动关联。
- 手册版本、工单和询价数据仅保存在当前浏览器，没有账号、云同步和多人协作能力。
- 缺少自动化测试、持续集成、可访问性审计和系统化浏览器兼容验证。

## 后续开发建议

1. 优先深化产品模板和真实实验记录，为每项结论保留官方资料或验证证据。
2. 按客户场景扩展厂商研究，避免脱离型号、版本和授权条件进行简单参数对比。
3. 将产品、厂商、实验和方案数据从页面脚本中拆分为独立结构化数据文件。
4. 在需要跨设备或多人使用时，再引入后端存储、身份认证、权限和数据备份。
5. 为搜索、筛选、产品比较和本地数据操作增加自动化测试及 CI 检查。
6. 建立版本更新机制，定期核验厂商链接、产品状态、软件版本和授权信息。

## 本地运行

项目是静态站点，可直接打开 `index.html`。如需生成部署目录：

```bash
npm run build
```

构建结果输出到 `dist/`，该目录不会提交到 Git。

## 数据说明

站内产品与工程内容用于学习和方案研究。具体性能、接口、授权、兼容性、命令和配置路径应以对应厂商、型号及版本的最新官方资料为准。
