# IT Solution Lab

IT Solution Lab 是一个面向企业 IT 解决方案学习的个人技术实验室，用于持续整理网络、安全、云计算、产品研究、实验验证和方案设计内容。

- 在线预览：[https://it-solution-lab.vercel.app](https://it-solution-lab.vercel.app)
- GitHub：[coke-at/IT-Solution-Lab](https://github.com/coke-at/IT-Solution-Lab)

## 项目定位

项目记录一名售后工程师向安全售前方向发展的学习过程，重点展示知识如何经过产品理解、实验验证、方案组织和厂商选型，逐步形成可以检查和继续完善的能力证据。

它不是产品宣传页，也不代表已经完成的客户项目。当前建设目标包括：

- 建立网络、安全、云计算与解决方案设计的结构化学习路径。
- 使用统一模板理解产品解决的客户问题、核心能力和实践方向。
- 记录个人实验验证过程，为技术判断保留可复查的依据。
- 通过模板方案练习从客户问题到技术选型、架构设计和实施规划的表达。
- 以客观的场景匹配方式整理不同厂商的技术路线。

## 技术栈

- HTML5、CSS3、原生 JavaScript（ES Modules）
- Browser Local Storage，用于保存工单、询价和手册版本等本地数据
- Node.js 构建脚本，用于生成 `dist/client`、`dist/server` 和托管配置
- Fetch Worker 形式的静态资源服务入口
- Git / GitHub 版本管理

项目没有引入前端框架或第三方运行时依赖。

## 数据真实性说明

- 当前解决方案是学习用途的模板方案，状态为“正在完善”，不代表真实客户案例或已经完成的交付项目。
- 当前实验是个人学习与验证记录，验证项描述的是目标、检查方式和预期观察结果，不虚构实际测试结果。
- 厂商对比用于练习场景化选型，不提供排名、市场份额、价格结论或未经验证的性能比较。
- 产品内容不绑定具体型号和软件版本。性能、接口、授权、兼容性、命令及配置路径应以对应厂商的最新官方资料为准。
- 站点中的数量和状态由现有产品、实验、方案及厂商数据动态生成，不作为能力评分或项目成果证明。

## 当前能力边界

- 已建立产品学习、实验记录、模板方案和厂商分析的统一表达结构，但内容仍需持续补充官方资料与个人验证证据。
- “企业出口安全模拟”尚未形成完整配置、日志截图和实际验证结论。
- “成长型企业安全与云基础设施升级”尚未结合具体客户环境完成容量、授权、兼容性和实施边界核验。
- Dashboard 聚合已有内容和状态，不使用无依据的百分比或能力评分。
- 手册版本、工单和询价等数据仅保存在当前浏览器，没有账号、云同步和多人协作能力。
- 项目尚未建立完整的自动化测试、可访问性审计和系统化浏览器兼容验证。

## 部署方式

项目通过 GitHub 保存源码，并由 Vercel 从 `main` 分支自动构建和部署公开预览。

本地开发可直接打开 `index.html`。如需生成与部署一致的静态目录：

```bash
npm run build
```

构建结果输出到 `dist/`，该目录不会提交到 Git。Vercel 的构建与路由配置位于 `vercel.json`。
