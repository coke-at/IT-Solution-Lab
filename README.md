# IT Solution Lab

IT Solution Lab 是一个面向企业 IT 解决方案学习的个人技术实验室，记录一名售后工程师向安全售前方向发展的学习过程。项目围绕网络、安全、云计算和方案设计，将分散的学习内容组织成可持续完善的能力证据。

- 在线预览：[it-solution-lab.vercel.app](https://it-solution-lab.vercel.app)
- 源码仓库：[coke-at/IT-Solution-Lab](https://github.com/coke-at/IT-Solution-Lab)
- 当前性质：个人学习与作品集项目，非商业产品，非真实客户交付项目

## 为什么创建

售后工作更关注部署、排障和问题闭环，安全售前还需要理解客户需求、技术选型、方案架构与实施边界。这个项目用于把已有实践经验和新的学习内容连接起来，形成一条可以持续检查和补充的成长路径：

```text
技术学习 -> 产品理解 -> 实验验证 -> 厂商分析 -> 方案设计 -> 售前流程 -> 能力证据
```

项目不以堆积产品参数为目标，而是关注三个问题：产品解决什么客户问题、技术判断如何得到验证、不同能力如何组合成可实施的方案。

## 核心模块

| 模块 | 当前内容 | 主要作用 |
| --- | --- | --- |
| 首页 | 成长路线、学习方向、精选内容与能力证据入口 | 建立项目定位并连接各模块 |
| 技术地图 | 网络、安全、云计算、解决方案四个方向 | 整理学习顺序和当前学习状态 |
| 产品实验室 | AF、aTrust、EDR、SIP、HCI、AC 共 6 个产品模板 | 理解产品定位、客户问题、核心能力和售前关注点 |
| 实验中心 | “企业出口安全模拟”个人验证记录 | 记录实验目标、环境、验证项和复盘方法 |
| 厂商对比 | 深信服、华为、H3C 场景化分析 | 练习基于需求、技术路线和限制因素进行选型 |
| 解决方案中心 | “成长型企业安全与云基础设施升级”模板方案 | 练习需求分析、架构设计、产品组合和实施规划 |
| 成长 Dashboard | 技术、产品、实验、方案与选型证据聚合 | 展示能力从学习到验证和方案表达的形成过程 |
| 资源中心 | 设备知识库、用户手册、交付标准与问题排障 | 整理学习资料和工程参考内容 |
| 业务工具 | 工单、报价、询价和招投标工具 | 支撑本地工作记录与流程练习 |

各页面优先复用 `technologyDomains`、`productLabs`、`labExperiments`、`solutionCases` 和 `vendorComparisons` 等现有数据源，避免同一内容在多个模块中重复维护。

## 技术栈

- HTML5、CSS3、原生 JavaScript
- Hash 路由与数据驱动渲染
- 原生 `<dialog>`、ARIA 状态和键盘焦点支持
- Browser Local Storage，用于保存工单、询价和手册版本等本地数据
- Node.js 构建脚本，生成静态部署产物
- Git、GitHub 与 Vercel

项目没有引入前端框架或第三方运行时依赖。构建结果包含静态客户端资源、Fetch Worker 入口和托管配置。

## 数据真实性说明

- 当前解决方案是学习用途的模板方案，状态为“正在完善”，不代表真实客户案例、已交付项目或客户验收结果。
- 当前实验是个人学习与验证记录；验证项描述验证目标、检查方式和预期观察结果，不虚构实际测试结果。
- 厂商对比用于练习场景化选型，不提供厂商排名、市场份额、价格结论或未经验证的性能比较。
- 产品内容不绑定具体型号和软件版本。性能、接口、授权、兼容性、命令及配置路径应以厂商最新官方资料为准。
- 页面中的数量和状态来自现有数据聚合，不作为能力评分、项目收益或交付成果证明。

## 部署方式

项目由 GitHub 保存源码，Vercel 从 `main` 分支自动构建并部署公开预览。

本地可直接打开 `index.html` 查看站点。生成部署产物：

```bash
npm run build
```

构建结果输出到 `dist/`：

```text
dist/
├── client/   # 静态页面、样式、脚本和站点资源
├── server/   # Fetch Worker 入口
└── .openai/  # 托管配置
```

`dist/` 不提交到 Git；Vercel 的构建命令和输出目录配置位于 `vercel.json`。

## 当前能力边界

- 已建立产品学习、实验记录、模板方案和厂商分析的统一表达结构，但内容深度仍需通过官方资料和个人验证持续补充。
- “企业出口安全模拟”尚未形成完整配置记录、日志截图和实际验证结论。
- “成长型企业安全与云基础设施升级”尚未结合具体客户环境完成容量、授权、兼容性和实施边界核验。
- Dashboard 只聚合已有内容和状态，不使用无依据的百分比或能力评分。
- 手册版本、工单和询价等数据仅保存在当前浏览器，不提供账号、云同步或多人协作能力。
- 当前自动化测试和浏览器兼容验证仍有限，需要随功能演进继续补充。
